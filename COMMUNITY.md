# GitHub Community Standards

## 🌟 Our Community Values

YOZ-LMS is built on the foundation of collaboration, respect, and innovation. We welcome contributors from all backgrounds and experience levels.

### Core Values

- **🤝 Inclusivity**: Everyone is welcome, regardless of background, experience, or identity
- **🎯 Excellence**: We strive for high-quality code and documentation
- **🧩 Collaboration**: We work together to solve problems and build amazing features
- **📚 Learning**: We encourage continuous learning and knowledge sharing
- **🔍 Transparency**: We maintain open and honest communication

## 📋 Community Guidelines

### ✅ Do

- **Be respectful and professional** in all interactions
- **Welcome newcomers** and help them get started
- **Provide constructive feedback** on code and ideas
- **Follow the code of conduct** in all community spaces
- **Search before asking** to avoid duplicate questions
- **Use clear and descriptive titles** for issues and PRs
- **Document your changes** to help others understand your work
- **Test your changes** before submitting pull requests

### ❌ Don't

- **Use disrespectful or offensive language**
- **Harass or discriminate** against any community member
- **Spam or self-promote** excessively
- **Submit low-quality or incomplete pull requests**
- **Ignore feedback** from maintainers or community members
- **Post security vulnerabilities** publicly (use security@yoz-lms.com)
- **Create duplicate issues** without checking existing ones
- **Demand immediate responses** from maintainers
